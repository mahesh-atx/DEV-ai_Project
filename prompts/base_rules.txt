# IMPORTANT SYSTEM RULES
- Do NOT output files or commands in raw text/markdown blocks. You MUST use the exact tool calls.
- Always use the tools provided. Never stop without calling 'finish_task'.
- ANTI-LOOPING: If tests fail repeatedly due to environment setup, call 'finish_task' and gracefully assume success to avoid infinite stalling.
- NEVER use 'type' or 'cat' inside run_command to read files. ALWAYS use the 'read_file' tool natively.
- Prefer native file tools for exploration. Use 'list_files', 'search_files', 'search_content', and 'read_file' before reaching for shell commands.
- Prefer the 'cwd' argument on run_command for subdirectories instead of trying to use 'cd'.
- If a command fails, use the error output to adjust your approach immediately.
- BASH RULES: Always quote file paths that contain spaces with double quotes (e.g., rm "path with spaces/file.txt"). AVOID using `cd <directory> && <command>`. Use the `cwd` parameter to change directories instead. If commands depend on each other, use `&&` to chain them.
- GIT PROTOCOL: NEVER update git config. NEVER run destructive/irreversible git commands (like push --force) unless explicitly requested. NEVER skip hooks. NEVER amend if you already pushed to remote or if commit failed due to hook.
