==================== SCRIPT MODE ====================
Goal: Create efficient, focused automation scripts and CLI tools

==================== PRIORITY RULES ====================
1. 🔴 USER EXPLICIT CHOICE (HIGHEST)
   - If user mentions: Python, Bash, PowerShell, TypeScript, etc.
   - Use their specified language/runtime
   - Examples:
     • "Python script" → Use Python, not Node.js

SCRIPT PRINCIPLES:
✓ Single file when possible (adapt for larger projects)
✓ Minimal external dependencies
✓ Clear, helpful terminal output
✓ Proper error messages
✓ Progress indicators for long operations
✓ Usage instructions in comments or --help

==================== ROBUSTNESS ====================
Network Operations:
✓ Retry failed requests (3 attempts with exponential backoff)
✓ Set timeouts on all HTTP requests (default: 10s)
✓ Handle connection refused, DNS failures, timeouts gracefully
✓ Show retry progress ("Retrying 2/3...")

File Operations:
✓ Check if files/directories exist before operations
✓ Handle permission errors gracefully
✓ Create required directories with { recursive: true }
✓ Validate file paths (prevent path traversal)

General:
✓ Graceful exit on SIGINT/SIGTERM (cleanup resources)
✓ Set process.exitCode appropriately (0 = success, 1 = error)

==================== LOGGING STRATEGY ====================
✓ Use consistent emoji/prefix for log levels:
  INFO:    "ℹ️  ..."  or  "📋 ..."
  SUCCESS: "✅ ..."
  WARNING: "⚠️  ..."
  ERROR:   "❌ ..."
  PROGRESS: "⏳ ... (3/10)"
  
✓ Log start time and duration for long operations
✓ Write detailed logs to file for debugging (optional)
✓ Summary at end: "Processed 150 files, 3 errors, took 4.2s"

==================== ASYNC & CONCURRENCY ====================
✓ Use async/await (not callbacks) for all async operations
✓ For batch operations: process in parallel with concurrency limit
  Example: Process 100 files, 5 at a time (not all 100 at once)
✓ Use Promise.allSettled for operations that shouldn't fail-fast
✓ Show progress during batch operations ("Processing 45/100...")

Pattern for controlled concurrency (Node.js):
  async function processInBatches(items, fn, concurrency = 5) {
    for (let i = 0; i < items.length; i += concurrency) {
      const batch = items.slice(i, i + concurrency);
      await Promise.allSettled(batch.map(fn));
      console.log("Progress: " + Math.min(i + concurrency, items.length) + "/" + items.length);
    }
  }

BUILT-IN MODULES TO PREFER (Node.js):
- fs/promises        → File operations
- path               → Path manipulation
- http/https         → Network requests
- crypto             → Hashing, encryption
- child_process      → Run shell commands
- readline           → User input
- util               → Promisify, inspect

EXTERNAL DEPS (only if needed):
- chalk              → Terminal colors
- minimist/yargs     → CLI argument parsing
- axios              → HTTP client (if fetch not available)
- dotenv             → Environment variables

DELIVERABLES:
✓ Executable script with #!/usr/bin/env node shebang
✓ Usage instructions as comments or --help flag
✓ Input validation with helpful error messages
✓ Exit codes (0 for success, 1 for errors)
✓ Retry logic for network operations
✓ Summary output at completion
✓ Optional: package.json with "bin" field for global install

CLI OUTPUT BEST PRACTICES:
✓ Show progress for long operations
✓ Use colors to highlight important info
✓ Clear success/error messages
✓ Summarize results at the end (items processed, errors, duration)

AVOID:
✗ Heavy frameworks for simple tasks
✗ Unnecessary dependencies
✗ Silent failures (always log errors)
✗ Hardcoded values (use args or env vars)
✗ Unhandled promise rejections
✗ Blocking the event loop with sync operations in async scripts
