# Refactor MainMenu with Modal Popups

## Summary
Replace separate navigation screens for Model/Mode/Settings selection with modal popup overlays on top of the MainMenu. Background dims when a popup is open. All popups use uniform width (56 chars).

## Architecture

Current flow: MainMenu → navigate to ModeSelect/ModelSelect/SettingsScreen (full screen replacement)
New flow: MainMenu stays visible → modal overlay appears on top with dimmed backdrop

## Changes Required

### 1. Create `tui/components/Modal.jsx`
- Reusable modal wrapper component
- Props: `isOpen`, `onClose`, `title`, `children`
- Renders a dimmed backdrop over the full terminal
- Centers a bordered box with title
- Handles Escape key to close via `useInput`

### 2. Refactor `tui/components/MainMenu.jsx`
- Add state: `activeModal` ('none' | 'modes' | 'models' | 'settings')
- Keep MainMenu always visible as background
- When user selects "Change Mode", "Change Model", or "Settings" → set `activeModal` state
- Render `<Modal>` overlay with the appropriate selector inside
- Background dims via Modal's backdrop
- On selection or Escape → close modal, apply selection, return to MainMenu

### 3. Update `tui/components/ModelSelect.jsx`
- Remove `onBack` prop (modal handles closing)
- Add `onClose` prop
- Change width from 92 to 56 for uniform sizing

### 4. Update `tui/components/ModeSelect.jsx`
- Remove `onBack` prop (modal handles closing)
- Add `onClose` prop
- Change width from 72 to 56 for uniform sizing

### 5. Update `tui/components/SettingsScreen.jsx`
- Remove `onBack` prop (modal handles closing)
- Add `onClose` prop
- Change width from 72 to 56 for uniform sizing
- Keep internal menu/change-key screens working inside modal

### 6. Update `tui/app.jsx`
- Remove `modeSelect` and `modelSelect` views
- MainMenu now handles its own modals internally
- MainMenu passes `onModeChange` and `onModelChange` callbacks to App
- App only needs: setup, welcome (MainMenu), chat
- Settings modal is fully contained within MainMenu's modal system

## Implementation Order
0. Push all current changes to git (user requested)
1. Create Modal.jsx component
2. Update ModelSelect.jsx (uniform width 56, onClose prop)
3. Update ModeSelect.jsx (uniform width 56, onClose prop)
4. Update SettingsScreen.jsx (uniform width 56, onClose prop)
5. Refactor MainMenu.jsx (add modal state, render overlays, handle callbacks)
6. Update app.jsx (remove separate mode/model/settings views, simplify)
